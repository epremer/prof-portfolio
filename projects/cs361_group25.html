Lizz Learns Programming

CS361 Group Project: Farm2U
10 June 2018
Description
Our Customer created a vision statement for his CS361 individual project proposal and it was assigned to us. The general idea is likened to an "Uber for 'X'"-type software application that allows Farmers to list, manage, and sell their produce to their customers. When purchased, a third party aggregate worker (much like the driver in a literal Uber situation) - called a "Rabbit" - would pick up the produce and deliver it to the paying customer.
Preparation
The purpose of this project is to teach us various development lifecycle methods and their respective documents requirements, etc. The idea is that in fulfilling the requirements called for by Waterfall - at first - then switching to Spiral, and lastly the Agile Method for the final two assignments, we would learn the necessary skills.
Any Snags?
There are bugs that we would have fixed had we had more weeks to work on the project, but it was expected - required in fact - that we not submit a completed project.
Programming Environment
My group wrote a website application using JavaScript, Nodejs, Express, Handlebars, and the usual HTML and CSS. For version control, we used GitHub. For our database, we used phpMyAdmin through OSU. We used node.js on OSU's FLIP server to run the site. My experiences with the project can be found here.
