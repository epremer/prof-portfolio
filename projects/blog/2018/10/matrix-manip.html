Lizz Learns Programming

Matrix Manipulation
9 November 2018
Description
The purpose of this project was to get me familiar with the Unix shell, shell programming, Unix utilities, standard input, output, error, pipelines, process ids, exit values, and signals (at a basic level). It was an assignment for OSU's CS344 Operating Systems I course that I found difficult but really fulfilling and fun. This assignment was programmed in bash scripting language for CentOS, using our own dedicated server, since this class can sometimes crash the server. Yikes!
Preparation
I did not have much of a knowledge of bash scripting, so I did need to watch many lectures on bash scripting syntax. Thankfully, I had/have a great baseline of knowledge on command line programming, as I frequently work in Vim on MacOS Terminal, and I use Terminal for testing my programs.
Any Snags?
Looking back at this assignment after about a month, this project seems so straight-foward and simple (not "easy") that it feels silly how much difficulty I had in creating it. I guess that's the nature of the beast; you have to keep learning, keep improving, which means when you look back at previous projects, they really should seem simple in retrospect.

I do recall that the syntax of bash was difficult to grasp at first. It's unique in its workings; different than C or C++, for sure.
Programming Environment
I developed this bash script on the OSU school server, os1, which runs CentOS. I used Vim on MacOS Terminal.
