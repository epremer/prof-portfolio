Lizz Learns Programming

Text-Based Adventure Game
1 November 2018
Description
This project was two C programs to create a text-based game that takes the user through various rooms until they get to the end room, which is randomly chosen by the program.

The first of the two programs creates a series of files within a new directory (unique to that process) that hold descriptions of the in-game rooms and how they are connected. The second program uses the rooms created in the most recent run of the room building program, randomly selects a start room and an end room, and leads the user through the rooms (the user chooses the room traversal) until the end room has been reached.
Assignment Specifications
Room Name:
A room name cannot be assigned to more than one room. Each name can be at max 8 characters long, case matters. Must have 10 hardcoded room names from which the program chooses 7.

Room Type:
Possible types are: START_ROOM, END_ROOM, and MID_ROOM. Type is randomly assigned, however there can only be one start room and one end room.

Outbound Connections to other rooms:
There must be between 3 and 6 connections to and from each room. Outbound connections must have matching connections coming back. A room cannot have an outbound connection that points to itself. A room cannot have more than one outbound connection to the same room.

Room files must contain: 
ROOM NAME:
CONNECTION 1: 
CONNECTION 2: 
... 
ROOM TYPE:


An example room name would be: 

ROOM NAME: XYZZY
							
CONNECTION 1: PLOVER
							
CONNECTION 2: Dungeon
							
CONNECTION 3: twisty
							
ROOM TYPE: START_ROOM

When playing the game, the current room and its connections should be displayed as: 

CURRENT LOCATION: XYZZY
							
POSSIBLE CONNECTIONS: PLOVER, Dungeon, twisty.
							
WHERE TO? >

Input validation: Should the player type a next room name that does not exactly match a room in the connections list, an error should be shown and the player should be reprompted for a new room.
Any Snags?
I was fortunate enough to have help from the course teaching assistants and the instructor
Programming Environment
I developed this bash script on the OSU school server, os1, which runs CentOS. I used Vim on MacOS Terminal on Linix.
